# TPL-030 — Tuần đầu — Checklist 5 ngày

**Buổi sử dụng:** B10 (Sẵn sàng vào dự án)
**Format gợi ý:** Markdown / In ra giấy

## Mục đích
Lịch chi tiết tuần đầu của BA fresher — không trôi mất ngày nào.

## Template

```markdown
# Tuần đầu — Checklist 5 ngày

**Tên:** _______________
**Công ty:** _______________
**Ngày bắt đầu:** ___/___/___

---

## NGÀY 1 — Setup + Chào hỏi

### Buổi sáng (8h-12h)
- [ ] Đến công ty đúng giờ (sớm 15 phút)
- [ ] HR onboarding (giấy tờ, badge, laptop)
- [ ] Setup laptop:
  - [ ] VS Code
  - [ ] Claude Code extension
  - [ ] Slack / Teams
  - [ ] Email công ty
  - [ ] VPN (nếu cần)
  - [ ] Trình duyệt + bookmark internal tools
- [ ] Ăn trưa với team (đừng ăn 1 mình)

### Buổi chiều (13h-17h)
- [ ] Manager 1-1 đầu tiên (30-60 phút)
  - Hỏi: kỳ vọng 30/60/90, kênh báo cáo, tools
- [ ] Tự intro với team (Slack post / standup)
- [ ] Hiểu org chart (vẽ ra giấy)
- [ ] Tour văn phòng (toilet, pantry, meeting room)
- [ ] Ngày 1 reflection (5 phút trước về): "Hôm nay học được gì?"

---

## NGÀY 2 — Đọc + Hỏi

### Buổi sáng
- [ ] Setup tools còn lại:
  - [ ] Jira (request access nếu chưa có)
  - [ ] Confluence (browse spaces)
  - [ ] Figma / design tool
  - [ ] Database client (nếu cần xem data)
- [ ] Đọc tài liệu cũ:
  - [ ] 1 BRD
  - [ ] 1 SRS (gần nhất)
  - [ ] 1 Confluence wiki onboarding (nếu có)

### Buổi chiều
- [ ] List 20 câu hỏi (xem TPL-031)
- [ ] Schedule mentor 1-1 (chiều mai hoặc thứ 6)
- [ ] Tham dự 1 meeting (im lặng quan sát)

---

## NGÀY 3 — Quan sát meeting

### Buổi sáng
- [ ] Tham dự daily stand-up (im lặng, note format)
- [ ] Đọc thêm tài liệu (mỗi ngày 2-3 file)
- [ ] Note câu hỏi thắc mắc

### Buổi chiều
- [ ] Tham dự 1-2 meeting nữa
- [ ] Mentor 1-1 (đặt 5-10 câu hỏi từ list 20 câu)
- [ ] Hỏi mentor: "Pattern viết tài liệu team đang theo?"

---

## NGÀY 4 — Hiểu workflow

### Buổi sáng
- [ ] Đi qua workflow Jira chuẩn (status, transition)
- [ ] Tự tạo 1 ticket test (low-priority hoặc tài liệu)
- [ ] Học format SRS team đang dùng

### Buổi chiều
- [ ] Sit-with 1 dev (30 phút) — quan sát họ làm việc
- [ ] Sit-with 1 QA (30 phút) — hiểu test workflow
- [ ] Hỏi dev/QA câu hỏi từ TPL-031

---

## NGÀY 5 — Đề xuất 1-1 với manager

### Buổi sáng
- [ ] Tham dự sprint review (nếu có) hoặc retrospective
- [ ] Note observations về team dynamics

### Buổi chiều
- [ ] Manager 1-1 (cuối tuần)
  - Confirm hiểu đúng kỳ vọng 30/60/90
  - Hỏi success metrics
  - Hỏi resource available (mentor, training, courses)
  - Đề xuất kế hoạch tuần 2
- [ ] Tổng kết tuần 1: viết note 1 trang
  - Đã học được gì?
  - Đã gặp ai?
  - Đã đọc tài liệu nào?
  - Câu hỏi còn lại?
- [ ] Chia sẻ note với mentor + manager
```

## Sample reflection note cuối tuần 1

```markdown
# Tuần 1 Reflection — Nguyễn Văn A

**Ngày:** 01-05/03/2026

## Đã học được

- Hiểu công ty làm về SaaS B2B HR cho SME Việt Nam
- Workflow team: 2-week sprint, daily 9h, demo cuối thứ 6
- Format SRS team dùng template internal (khác BABOK chuẩn)
- Tool stack: Jira, Confluence, Figma, Notion, Slack

## Đã gặp ai

- 12 người trong team mở rộng (BA + Dev + QA + PM + Designer + DevOps)
- Manager: Trần Thị B (BA Lead)
- Mentor: Phạm Văn C (Senior BA, 5 năm kinh nghiệm)

## Đã đọc

- 1 BRD (Phân hệ Tuyển dụng)
- 1 SRS (Phân hệ Chấm công — 45 trang)
- 3 Confluence wiki onboarding
- 5 user story trong Jira

## Câu hỏi còn lại

- Quy trình code review của team như nào?
- Có tools nào đặc biệt cho BA team đang dùng không?
- Khi nào em có thể start 1 task standalone?

## Plan tuần 2

- Shadow mentor trong sprint refinement
- Viết User Story đầu tiên (low priority để practice)
- Xin tham gia 1 phân hệ nhỏ
```

## Tips

- **Đừng push deliver gì lớn tuần đầu** — focus vào absorb
- **Note real-time** — Notion / Markdown
- **Ăn trưa với người khác mỗi ngày** — build network từ ngày 1
- **Hỏi nhiều câu mở** — không "có/không"
- **Đừng sợ "trông ngu"** — fresher hỏi nhiều mới đúng
- **Lưu reflection note** — sau 90 ngày so sánh, thấy tiến bộ rõ
